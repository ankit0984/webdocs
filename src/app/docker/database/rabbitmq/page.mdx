# RabbitMQ with Docker Compose

This guide demonstrates how to run **Bitnami RabbitMQ** in Docker Compose.
The Bitnami RabbitMQ image is **production-ready**, **secure by default**, and supports **plugins and clustering**.

---

## Docker Compose File

Below is the configuration for running RabbitMQ:

```yaml filename="docker-compose.yml"
services:
  rabbitmq:
    image: docker.io/bitnami/rabbitmq:4.1
    ports:
      - '15672:15672'
    environment:
      - RABBITMQ_SECURE_PASSWORD=yes
      - RABBITMQ_LOGS=-
      - RABBITMQ_USERNAME=${RABBITMQ_USERNAME}
      - RABBITMQ_PASSWORD=${RABBITMQ_PASSWORD}
      - RABBITMQ_ERL_COOKIE=${RABBITMQ_ERL_COOKIE}

    volumes:
      - ./rabbitmq_data:/bitnami/rabbitmq/mnesia
volumes:
  rabbitmq_data:
    driver: local
```
### Configuration file
create .env file for setting variables

```dotenv filename=".env"
RABBITMQ_USERNAME=root
RABBITMQ_PASSWORD=secretpassword
RABBITMQ_ERL_COOKIE=devuser
```
