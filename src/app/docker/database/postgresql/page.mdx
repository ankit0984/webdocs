# Bitnami PostgreSQL Setup with Docker Compose

This guide demonstrates how to run **Bitnami PostgreSQL** using Docker Compose.
The Bitnami image is production-ready, secure by default, and easy to configure.

## Why use bitnami PostgreSQL?:
Bitnami provides pre-packaged and well-maintained container images that are widely trusted in the industry.
Here are some key advantages of using the Bitnami PostgreSQL Docker image over other PostgreSQL images:

1. Production-Ready – Bitnami’s images are hardened for security and optimized for performance.
2. Regular Updates – Images are frequently updated to include security patches and the latest versions of PostgreSQL.
3. Environment Variable Configuration – Supports easy configuration of root user, database creation, and user credentials via environment variables.
4. Cross-Platform Compatibility – Works seamlessly across Linux, macOS, and Windows environments.
5. Data Persistence – Comes preconfigured with volume mounting for persistent storage of PostgreSQL data.
6. Community & Enterprise Support – Backed by VMware and trusted in enterprise-grade environments.
7. Lightweight & Secure – Optimized with minimal vulnerabilities compared to unofficial images.

## Container file
`docker.compose.yml` file for creating PostgreSQL docker image

```yaml filename="docker.compose.yaml"
version: "3.8"

services:
  postgresql:
    container_name: container_name
    image: docker.io/bitnami/postgresql:latest
    environment:
      - POSTGRESQL_ROOT_USER=${POSTGRESQL_ROOT_USER}
      - POSTGRESQL_ROOT_PASSWORD=${POSTGRESQL_ROOT_PASSWORD}
      - POSTGRESQL_USERNAME=${POSTGRESQL_USERNAME}
      - POSTGRESQL_PASSWORD=${POSTGRESQL_PASSWORD}
      - POSTGRESQL_DATABASE=${POSTGRESQL_DATABASE}

    ports:
      - "5432:5432"
    volumes:
      - "./pg_docker/postgresql_data:/bitnami/postgresql"
    restart: always


volumes:
  postgresql_data:
    driver: local
```
### Configuration file
create .env file for setting variables

```dotenv filename=".env"
POSTGRESQL_ROOT_USER=root
POSTGRESQL_ROOT_PASSWORD=secretpassword
POSTGRESQL_USERNAME=devuser
POSTGRESQL_PASSWORD=devpass
POSTGRESQL_DATABASE=mydb
```
### Run Docker Compose:
```bash
$ docker compose up -d
```
